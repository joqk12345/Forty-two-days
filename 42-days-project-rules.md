# 《42天》项目写作规则

参考来源：`/Users/mac/Documents/workspace/codespace/token-to-serving/agents.md`

这份规则把非虚构写作中的严谨性，转化为历史小说项目的创作约束。目标不是把小说写成论文，而是让小说在史实、因果、人物选择和叙事推进上更可靠。

## 一、核心承诺

《42天》讲的是王阳明在宁王朱宸濠之乱中，如何在信息不全、兵力不足、朝廷命令未至、地方官摇摆的情况下，用名分、疑兵、反间、调兵、反攻南昌和鄱阳湖决战，一步步压缩朱宸濠的选择空间。

它不是王阳明传奇爽文，不是心学语录拼贴，也不是流水账式历史复述。

## 二、目标读者

目标读者是对历史、权谋、军事谋略、王阳明心学感兴趣的普通读者。

读者不需要熟悉明代制度，但希望看到：

- 事为什么会发生。
- 人为什么这样选。
- 王阳明到底高在哪里。
- 朱宸濠为什么一步步失去主动。
- 心学如何体现在行动里，而不是口号里。

## 三、五条最高规则

### 1. 局势问题先行

每一章都要从一个具体局势问题开始，而不是从历史背景说明开始。

例如：

- 王府门为什么忽然落闩？
- 城门为什么只让文书出，不让人出？
- 王阳明为什么不先关丰城？
- 宁王为什么迟迟不敢离开江西？
- 安庆告急时，为什么反攻南昌才是关键？

### 2. 史实证据先于顺口叙述

不能为了情节顺滑，把史实写得过度确定。

如果史料明确，就按明确写。

如果史料有争议，就用小说方式避开精确断言，例如：

- 用“第几天”代替具体公历日期。
- 用“号称十万”而不是硬写实数。
- 用“据传”“来报”“檄文称”区分消息层级。

### 3. 条件必须跟着判断走

凡是涉及兵力、时间、路线、速度、战术判断，都要写清楚条件。

不要只写：

```text
王阳明判断宁王会攻南京。
```

要写成：

```text
京师远，江西未定，粮道未稳；据江西只能坐等朝廷合围。南京近，有名、有仓、有江防，所以最险。
```

小说里不用写成论文，但人物判断必须有条件支撑。

### 4. 机制先于评价

少写“王阳明神机妙算”，多写他如何做到：

- 抢解释权。
- 让地方官不能装糊涂。
- 让宁王怀疑后方。
- 让正确建议在宁王帐中失效。
- 让宁王从主动东下变成被迫回师。

读者看懂机制，自然会觉得高明。

### 5. 每个阶段要可交接

每一个阶段完成后，要留下清楚的状态：

- 当前局势是什么？
- 谁掌握主动？
- 哪个问题已经解决？
- 哪个问题被推到下一章？

这能避免章节漂亮但主线松散。

## 四、每章默认问题

写每一章前，先回答七个问题：

1. 这一章解决什么局势问题？
2. 上一章的哪个因果导致这一章？
3. 这一章谁必须做选择？
4. 这个选择的条件是什么？
5. 哪个物件承载本章推进？
6. 这一章纠正读者什么误解？
7. 结尾把哪个问题推给下一章？

示例：第8章“三策”

- 问题：宁王到底可能怎么走？
- 因果：王阳明确认南昌有变后，必须判断危险方向。
- 选择者：王阳明。
- 条件：京师远、南京近、江西自守会被合围。
- 物件：地图、三枚棋子。
- 误解：王阳明不是猜中南京，而是压缩宁王选择空间。
- 下一章：如何用檄文抢名分。

## 五、证据等级

为了历史小说写作，采用四级证据：

- A：原典、正史、可靠年谱、奏疏、碑文、权威校注本。
- B：严肃历史研究、可靠传记、学术文章、带出处的资料整理。
- C：讲座转写、通俗文章、百科、未核验笔记。
- D：传闻、短视频口播、无出处说法。

使用原则：

- 大事件必须至少有 A/B 支撑。
- 细节场景可以用 C 启发，但不能作为硬史实。
- D 不能作为事实锚点，只能作为待核查线索。
- `029.md` 目前归为 C+：它的战略解释很有启发，但具体名词、日期、人物需回到 A/B 资料校准。

## 六、小说中的“引用”方式

正文不放学术引用。

但创作资料中要保留来源：

- 哪个史实来自《明史》。
- 哪个情节来自《王阳明全集·年谱》。
- 哪个解释来自 `029.md`。
- 哪个细节是虚构。

建议在后续建立 `source-ledger`，为重要事实建卡。

## 七、风格迁移

从非虚构规则中迁移到小说的风格要求：

- 精确：判断有条件，事件有因果。
- 具体：用物件和动作，不用空泛评价。
- 耐心：复杂战略一步步铺，不跳结论。
- 克制：不煽情，不喊口号。
- 可复查：关键史实在资料文件中能找到出处。

## 八、禁忌

- 不写王阳明开天眼。
- 不写朱宸濠纯粹愚蠢。
- 不把心学写成课堂讲义。
- 不让史实为爽点让路。
- 不让每章都变成谋略说明。
- 不让虚构人物改写历史结果。
- 不在古人对白里使用现代分析词。

## 十、每个阶段的交付块

每完成一组章节，可以用以下格式做自检：

```text
Owner: 当前执笔/修订者
Purpose: 本阶段解决的叙事问题
Evidence grade: 本阶段主要史实依据等级
Assumptions: 做了哪些艺术化假设
Open questions: 哪些史实还要查
Handoff: 下一阶段该写什么
```

这不是写进小说正文，而是保证项目长期推进时不丢线索。
